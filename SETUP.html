2 Enable BIOS Virtualisation

Required for Docker & WSL2 : the #1 reason for setup failures

Docker and WSL2 will not work without BIOS virtualisation enabled. Check this first!

How to check: Press Ctrl+Shift+Esc → Performance tab → CPU → look for "Virtualisation: Enabled".

If it says Disabled, you need to enter BIOS:

  1. Restart your PC
  2. During boot, press the BIOS key repeatedly:
Brand Key
Dell F2
HP F10 / Esc
Lenovo F1 / F2
ASUS / MSI F2 / Del
Acer F2 / Del
  1. Find Advanced → CPU Configuration
  2. Intel: Enable "VT-x" · AMD: Enable "SVM Mode"
  3. Save & Exit (F10)

If Task Manager already shows "Virtualisation: Enabled", you're good : click Next.

3 Install WSL2

Windows Subsystem for Linux : needed by Docker Desktop

Open PowerShell as Administrator and run the commands below.

How to open PowerShell as Administrator:

1. Press the Windows key
2. Type PowerShell
3. Right-click "Windows PowerShell"
4. Click "Run as administrator"
5. Click Yes on the popup
wsl --install

Restart your PC when prompted. Then open PowerShell (admin) again:

wsl --update
wsl --set-default-version 2

Verify (in a new terminal):

wsl --version
✅ You should see something like:WSL version: 2.x.x.x Kernel version: 5.15.167.4-1 WSLg version: 1.0.65

“wsl is not recognized” : WSL isn't installed. Run wsl --install as Administrator, then restart your PC.

Version shows 1.x : Run: wsl --set-default-version 2

wsl --install fails : Try enabling features manually:

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

Then restart and run wsl --set-default-version 2.

4 Install Git

Version control for the Supervisor's code checkpoints

Download Git for Windows

Run the installer : accept all defaults, just click Next through every screen.

Or install via terminal. Open PowerShell (Win key → type PowerShell → click it) and run:

winget install Git.Git --accept-source-agreements --accept-package-agreements

Verify (in a new terminal):

git --version
✅ You should see something like:git version 2.53.0.windows.1

“git is not recognized” : Close your terminal and open a new one. Git's PATH needs a new window to take effect.

Still not working? : Reinstall Git: download from git-scm.com and check "Add to PATH" during install.

5 Install Python 3.14

The Supervisor's core runtime

Download Python
CRITICAL: On the installer's first screen, check the box "Add python.exe to PATH". If you miss this, nothing will work!

Click "Install Now" → wait → Close.

Or install via terminal. Open PowerShell (Win key → type PowerShell → click it) and run:

winget install Python.Python.3.14 --accept-source-agreements --accept-package-agreements

Verify (open a new terminal : close the old one first!):

python --version
✅ You should see something like:Python 3.14.3
pip --version
✅ You should see:pip 24.3.1 from ... (python 3.13)

“python is not recognized” : You missed the "Add python.exe to PATH" checkbox during install. Uninstall Python from Settings → Apps, then reinstall and CHECK that box on the first screen.

Shows Python 3.12 or older : That's usually fine. Python 3.10+ works.

“pip is not recognized” : Try: python -m pip --version instead. If that works, use python -m pip in place of pip for all commands.

6 Install Node.js

Needed for the Gemini CLI and sandbox tooling

Download Node.js LTS

Download the LTS version (20.x or 22.x). Run installer, accept defaults.

Or install via terminal. Open PowerShell (Win key → type PowerShell → click it) and run:

winget install OpenJS.NodeJS.LTS --accept-source-agreements --accept-package-agreements

Verify (in a new terminal):

node --version
✅ You should see something like:v25.7.0 (or v24.x LTS)
npm --version
✅ You should see something like:11.11.0 (any recent version is fine)

“node is not recognized” : Close your terminal and open a new one. If still failing, restart your PC.

Version is below v18 : Uninstall via Settings → Apps, then re-download the LTS from nodejs.org.

7 Install Docker Desktop

Sandboxed containers for safe code execution

Download Docker Desktop
  1. Run Docker Desktop Installer.exe
  2. ✅ Check "Use WSL 2 instead of Hyper-V"
  3. Click OK → install → restart PC
  4. After reboot, launch Docker Desktop from Start menu
  5. Accept the agreement · skip sign-in (not required)
  6. Wait for the whale icon to stop animating
Docker Desktop must be running every time you use Supervisor. Auto-start: Docker Desktop → ⚙️ Settings → General → "Start when you sign in"

Verify:

docker --version
✅ You should see something like:Docker version 27.x.x, build bde2b89
docker info
✅ You should see something like (first few lines):Client: Docker Engine : Community Server: Containers: 0 Running: 0

“docker is not recognized” : Docker Desktop isn't installed, or you need to open a new terminal window.

“error during connect” or “daemon is not running” : Open Docker Desktop from the Start menu and wait for the whale icon to stop animating. Docker must be running every time you use Supervisor.

Tip: Set Docker to start automatically: Docker Desktop → ⚙️ Settings → General → "Start when you sign in"

8 Install & Authenticate Gemini CLI

The AI brain : sends prompts to Google's Gemini models

Install: Open PowerShell (Win key → type PowerShell → click it) and run:

npm install -g @google/gemini-cli

Authenticate: Run gemini, select "Login with Google", sign in with your Google account.

gemini
✅ You should see something like:? Login with Google (Use arrow keys) ❯ Login with Google Use an API Key

A browser will open → sign in → grant permissions → return to terminal → press Ctrl+C to exit.

“gemini is not recognized” : Gemini CLI isn't installed. Run npm install -g @google/gemini-cli again.

"npm is not recognized" : Node.js isn't installed correctly. Go back to Step 6.

Browser doesn't open : Check your default browser settings. Try running gemini again.

10 Project Setup

Install dependencies and build the sandbox

1. Make sure the folder exists:

C:\Users\YOUR_USERNAME\Desktop\Experiments\Antigravity Auto\

Replace YOUR_USERNAME with your Windows username.

2. Install Python packages. Open a terminal and navigate to the project folder:

How to navigate to the project folder:

1. Open PowerShell (Win key → type PowerShell → click it)
2. Type: cd "C:\Users\YOUR_USERNAME\Desktop\Experiments\Antigravity Auto"
3. Press Enter
pip install --upgrade -r requirements.txt

Verify:

python -c "import fastapi, uvicorn, aiohttp, psutil; print('All packages OK')"
✅ You should see:All packages OK

"No module named ..." : A package is missing. Run the pip install command again.

"pip is not recognized" : Try: python -m pip install --upgrade -r requirements.txt

Permission error : Add --user flag: pip install --user -r requirements.txt

3. Build the Docker sandbox image (3-10 min). Run from the same terminal:

cd supervisor && docker build -t supervisor-sandbox -f Dockerfile.sandbox .

Verify:

docker images supervisor-sandbox
✅ You should see something like:REPOSITORY TAG IMAGE ID CREATED SIZE supervisor-sandbox latest abc123def456 1 minute ago 850MB

"Cannot connect to the Docker daemon" : Docker Desktop isn't running. Open it from the Start menu and wait.

"Dockerfile not found" : Make sure you're in the project root folder. Run: dir supervisor\Dockerfile.sandbox

Network errors during build : Check your internet connection and try again.

4. Create a project folder under Experiments for each project, e.g.:

mkdir "C:\Users\YOUR_USERNAME\Desktop\Experiments\My Project"
